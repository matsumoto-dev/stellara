### 3.1. データ処理契約（DPA）— GDPR Art.28 準拠

> GDPR Art.28 に基づき、個人データの処理を委託する第三者（データプロセッサ）との間で、
> データ処理契約（Data Processing Agreement）の締結状況を開示する。

| プロセッサ | DPA 状況 | 備考 |
|-----------|---------|------|
| **Stripe** | 標準契約に DPA 条項を含む | [Stripe DPA](https://stripe.com/legal/dpa) |
| **Supabase** | 標準契約に DPA 条項を含む | [Supabase DPA](https://supabase.com/legal/dpa) |
| **Vercel** | 標準契約に DPA 条項を含む | [Vercel DPA](https://vercel.com/legal/dpa) |
| **Anthropic** | API 利用規約にデータ処理条項あり | 要確認: GDPR 準拠の DPA として十分か MVP 前に検証する |
| **Google** | 標準契約に DPA 条項を含む | [Google DPA](https://business.safety.google/processorterms/) |

**注意**: Anthropic の API 利用規約が GDPR Art.28 の全要件を満たすか、MVP 公開前に確認すること。不十分な場合は Anthropic に個別 DPA を要求するか、EU 居住者のデータを Claude API に送信しない代替設計を検討する。

### 3.2. データ侵害対応手順 — GDPR Art.33/34 準拠

**検知:**
- Supabase / Vercel のセキュリティアラートを監視する
- Stripe の Webhook 異常を監視する
- ユーザーからの不正アクセス報告を受け付ける

**72時間ルール（GDPR Art.33）:**
- 個人データの侵害を認知してから **72時間以内** に、該当するデータ保護当局（DPA）に通知する
- 通知内容: 侵害の性質、影響を受けるデータ主体の概数、想定される結果、講じた/講じる予定の対策

**ユーザー通知（GDPR Art.34）:**
- 侵害がユーザーの権利と自由に **高リスク** をもたらす場合、影響を受けるユーザーに速やかに通知する
- 通知手段: 登録メールアドレスへの直接通知 + アプリ内バナー
- 通知内容: 侵害の概要、想定される影響、ユーザーが取るべき対策（パスワード変更等）

**記録:**
- 全てのデータ侵害（通知の要否を問わず）を記録する
- 記録内容: 侵害の事実、影響、講じた対策、判断の根拠

**実装**: Phase 1 で侵害対応手順書を作成し、連絡先リスト（当局・プロセッサ）を準備する。

### 4. データ保持期間

| データ | 保持期間 | 削除トリガー |
|--------|---------|-------------|
| ユーザープロフィール | アカウント存続中 | アカウント削除 |
| 鑑定履歴 | アカウント存続中 | アカウント削除 |
| セッションログ | 90日 | 自動削除（cron） |
| 決済記録 | 税法に基づき7年 | 法的保持期間満了 |
| 匿名化統計データ | 無期限 | 個人識別不可のため削除対象外 |

**アカウント削除時:**
- ユーザーがアカウント削除をリクエストした場合、**当社の直接管理下にあるデータ**を **30日以内** に物理削除する
- 30日の猶予期間中は撤回可能（ソフトデリート状態）
- 削除完了後、通知メールを送信する
- **第三者プロセッサ（Anthropic）に以前送信されたデータ**は、各社の保持ポリシーに従う（Anthropic: 安全目的で最大30日間保持後に自動削除）
- 法的義務により保持が必要なデータ（税務記録）は、義務期間満了後に削除する

### 5. ユーザーの権利

| 権利 | 内容 | 対応方法 |
|------|------|---------|
| **アクセス権** | 自分のデータの閲覧 | アプリ内の「マイデータ」ページ（Phase 2） |
| **データポータビリティ** | データのエクスポート | JSON形式でダウンロード（Phase 2） |
| **削除権（消去権）** | アカウントと全データの削除 | アプリ内の「アカウント削除」ボタン（Phase 1） |
| **処理の制限** | 特定の処理への異議 | メールで申請 → 手動対応 |
| **同意の撤回** | いつでも同意を撤回可能 | アカウント削除で全同意を撤回 |

**対応期限**: リクエスト受領後 **30日以内** に対応（GDPR準拠）

### 6. Cookie ポリシー

**Phase 1 時点:**
- **使用する Cookie**: 認証セッション Cookie のみ（機能上必須）
- **トラッキング Cookie**: 使用しない
- **第三者 Cookie**: 使用しない
- **EU ePrivacy 指令の開示**: 本サービスは認証目的で厳密に必要なセッション Cookie を使用する。この Cookie はログイン状態の維持にのみ使用され、ブラウザを閉じると削除される。機能上必須の Cookie であるため同意バナーは不要だが、ここに使用を開示する

**Phase 2 以降（Analytics導入時）:**
- Google Analytics 等を導入する場合、Cookie 同意バナーを追加
- 必須 Cookie と任意 Cookie を分離
- このポリシーを更新する

### 8.1. カリフォルニア州居住者の権利 — CCPA/CPRA 準拠

> カリフォルニア消費者プライバシー法（CCPA）および CPRA に基づき、
> カリフォルニア州居住者には以下の追加的権利が付与される。

**知る権利（Right to Know）:**
- 過去12ヶ月間に収集した個人情報のカテゴリと具体的な項目
- 収集の出所、目的、共有先の第三者カテゴリ
- リクエスト方法: hello@stellara.chat に「CCPA Request」の件名で送信

**削除権（Right to Delete）:**
- 収集した個人情報の削除をリクエストできる
- 対応方法: §5「ユーザーの権利」の削除権と同一

**販売・共有のオプトアウト権（Right to Opt-Out）:**
- **{BRAND} はユーザーの個人情報を販売・共有しない**
- 「Do Not Sell or Share My Personal Information」: 該当する販売・共有行為なし

**非差別の権利（Right to Non-Discrimination）:**
- プライバシー権を行使したことを理由に、サービスの品質や価格で差別しない

**本人確認:**
- CCPA リクエストの処理にあたり、リクエスト者がアカウント所有者であることを確認する
- 確認方法: 登録メールアドレスからの送信 + アカウント情報の照合

**対応期限**: リクエスト受領後 **45日以内**（CCPA 準拠。15日間の延長が必要な場合は事前通知する）

### 9. セキュリティ対策

- 通信の暗号化（TLS/HTTPS）
- データベースの暗号化（Supabase のデフォルト暗号化）
- 認証: JWT + Supabase Auth
- Rate Limiting による不正アクセス防止
- 決済情報は Stripe が管理（PCI DSS準拠）。{BRAND} はカード番号を保持しない

### 10. ポリシーの変更

- 重要な変更は **30日前** にメールで通知する
- 軽微な変更（文言の明確化等）は事前通知なく行う場合がある
- 変更後の継続利用は、変更内容への同意とみなす
- 変更履歴はポリシーページに記載する

### 11. 連絡先

- データ保護に関する問い合わせ: hello@stellara.chat
- 応答目安: 3営業日以内
